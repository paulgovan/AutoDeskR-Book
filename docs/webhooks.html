10  Webhooks

Note

Coming in a future release. This chapter documents a planned addition to the AutoDeskR package. The API described here exists in APS but R wrapper functions are not yet available. Track progress at github.com/paulgovan/AutoDeskR.

11 Webhooks

The Webhooks API lets APS push HTTP notifications to a URL you control whenever a specified event occurs — for example, when a file translation completes, a new model version is published, or a construction issue changes status.

Using webhooks eliminates the need to poll checkFile() or checkPdf() in a loop. Instead, APS calls your server the moment the event fires.

11.1 Why It Matters

Polling works fine for interactive sessions, but in production pipelines it wastes API quota and adds latency. A webhook-driven pipeline looks like:

Upload file → POST /webhooks (register callback)
            ↓ (APS fires event when translation done)
     Your server receives POST → trigger downstream R job

This is especially useful for automated reporting pipelines where a new DWG published to ACC should automatically trigger translation, metadata extraction, and a Shiny dashboard refresh.

11.2 Supported Event Types

System Event
data dm.version.added — new file version created
data dm.version.modified — file version updated
derivative extraction.finished — Model Derivative job complete
derivative extraction.failed — Model Derivative job failed
autodesk.construction.issues issue.created — new issue created
autodesk.construction.issues issue.updated — issue status changed

11.3 Planned Functions

Function Description
createWebhook(token, system, event, callback_url, ...) Register a new webhook subscription
listWebhooks(token, system, event) List active subscriptions
getWebhook(token, hook_id) Retrieve a single subscription
updateWebhook(token, hook_id, ...) Update a webhook (e.g. change callback URL)
deleteWebhook(token, hook_id) Remove a subscription

11.4 Planned Workflow

# (Functions not yet implemented — illustrative only)
token <- getToken(id, secret, scope = "data:read data:write")

# Register: call my-server.example.com when any derivative extraction finishes
hook <- createWebhook(
  token        = token,
  system       = "derivative",
  event        = "extraction.finished",
  callback_url = "https://my-server.example.com/aps-hook",
  scope        = list(workflow = "my-pipeline")
)
hook_id <- hook$content$hookId

# Later — list active hooks
hooks <- listWebhooks(token, system = "derivative")

# Clean up
deleteWebhook(token, hook_id = hook_id)

Your callback server receives a POST with a JSON payload containing the event type, URN of the affected file, and a timestamp. Parse it and trigger downstream processing accordingly.
